# キーバインドガイド (KEYBINDINGS.md)

この環境は、VSCode から Neovim へのスムーズな移行をサポートするように設計されています。
キーバインドの多くは、VSCode のショートカットに対応するものが設定されています。

## 1. VSCode → Neovim 対応表

VSCode でよく使われる機能を Neovim でどのように実行するかをまとめました。

| 機能               | VSCode ショートカット | Neovim (Leader: Space) | 説明                                   |
| :----------------- | :-------------------- | :--------------------- | :------------------------------------- |
| ファイル検索       | `Ctrl+P`              | `<Space>ff`            | プロジェクト内のファイルを検索         |
| 全文検索           | `Ctrl+Shift+F`        | `<Space>sg`            | プロジェクト内のテキストを検索 (Grep)  |
| 定義へ移動         | `F12`                 | `gd`                   | 関数の定義元へジャンプ                 |
| 参照へ移動         | `Shift+F12`           | `gr`                   | 関数がどこで使われているかを一覧表示   |
| シンボル名変更     | `F2`                  | `<Space>cr`            | 変数や関数名を一括リネーム             |
| クイックフィックス | `Ctrl+.`              | `<Space>ca`            | コードアクションを表示                 |
| エクスプローラー   | `Ctrl+B`              | `<Space>e`             | ファイルツリーを開く (mini.files)      |
| 保存               | `Ctrl+S`              | `Ctrl+S`               | ファイルを保存                         |
| ターミナル切替     | `Ctrl+` (Backtick)    | `Ctrl+\`               | ターミナルの表示/非表示                |
| サイドバーを閉じる | `Ctrl+B`              | `q`                    | エクスプローラーなどで閉じるときに使用 |

## 2. Neovim 基本操作

Neovim の基本概念（モード）を理解しましょう。

### モードの切り替え
- **ノーマルモード**: 移動やコマンド入力。他のモードから `Esc` で戻ります。
- **挿入モード (Insert)**: テキスト入力。ノーマルモードで `i` を押すと入れます。
- **ビジュアルモード**: 範囲選択。ノーマルモードで `v` (文字単位) または `V` (行単位) を押します。

### 基本的な移動 (hjkl)
- `h`: 左
- `j`: 下
- `k`: 上
- `l`: 右

## 3. 主要キーマップ一覧

### Leader (Space) キーマップ
Leader キー（スペースキー）を起点とする強力なコマンド群です。

- **ファイル操作 (`<leader>f`)**
  - `<Space>ff`: ファイル検索 (Find files)
  - `<Space>fr`: 最近使ったファイル (Recent files)
  - `<Space>fb`: バッファ一覧 (Buffers)
  - `<Space>fn`: 新規ファイル作成

- **検索 (`<leader>s`)**
  - `<Space>sg`: プロジェクト内検索 (Live grep)
  - `<Space>sw`: カーソル下の単語を検索
  - `<Space>sk`: キーマップ一覧を表示
  - `<Space>sh`: ヘルプを検索
  - `<Space>sd`: 現在のファイルの診断結果 (Diagnostics)
  - `<Space>ss`: シンボル検索 (LSP symbols)

- **LSP / コード操作 (`<leader>c`)**
  - `gd`: 定義へ移動
  - `gr`: 参照を検索
  - `K`: ホバー情報表示 (ドキュメント)
  - `<Space>cr`: リネーム (Rename)
  - `<Space>ca`: コードアクション (Code action)
  - `<Space>cf`: フォーマット整形 (Format)

- **Git 操作 (`<leader>g`)**
  - `<Space>gg`: Lazygit を起動
  - `<Space>gs`: Git ステータスを表示 (Telescope)
  - `<Space>gc`: コミット履歴を表示
  - `]h` / `[h`: 次/前の変更箇所 (Hunk) へ移動
  - `<Space>ghp`: 変更箇所のプレビュー

- **デバッグ (`<leader>d`)**
  - `<Space>db`: ブレークポイント切り替え
  - `<Space>dc`: 実行/続行 (Continue)
  - `<Space>di`: ステップイン
  - `<Space>do`: ステップオーバー
  - `<Space>du`: デバッグ UI 切り替え

### ウィンドウ・バッファ操作
- `Ctrl+h/j/k/l`: ウィンドウ間の移動 (tmux 連携)
- `Shift+h/l`: 前/次のバッファに切り替え
- `<Space>|`: ウィンドウを垂直分割
- `<Space>-`: ウィンドウを水平分割
- `<Space>bd`: バッファを閉じる

## 4. tmux キーバインド

tmux の Prefix キーは **`Ctrl+Space`** に設定されています。

- `Ctrl+Space |`: ウィンドウを左右に分割
- `Ctrl+Space -`: ウィンドウを上下に分割
- `Ctrl+Space c`: 新規ウィンドウ作成
- `Ctrl+Space n/p`: 次/前のウィンドウへ移動
- `Ctrl+Space d`: デタッチ (tmux をバックグラウンドに残して抜ける)
- `Ctrl+h/j/k/l`: ペイン間を移動 (Neovim と共通)

---

## 5. デバッグ操作 (DAP)

1. `db` でブレークポイントを設置。
2. `dc` でデバッグを開始。
3. デバッグ UI が自動で開き、変数やスタックトレースを確認できます。
4. ステップ実行 (`di`, `do`) で一行ずつ確認。
5. `dt` でデバッグを終了。
